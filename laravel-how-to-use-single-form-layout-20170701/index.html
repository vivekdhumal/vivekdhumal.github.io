Laravel - How to use single form layout

July 1, 2017

Let's not create something new, just re-factor the existing things, also I assume that you have a good knowledge of Laravel framework (basic MVC).

well laravel has a great template engine called Blade, and it is much simpler and yet effective than any other template engines I have ever used. so as a laravel developer we come across the situation where we want to optimize our bulky form layouts to make it more organized and reusable. well laravel has already provision for that, you just need to figure it out how?.

Lets take the user model from laravel and consider that we are building the CRUD for users, in this situation do we required the separate form for create and update operations?. I don't think so, but having a two separate layout for each CRUD operation (create / update) it also acceptable solution, there is nothing wrong in that, but if we can use only one single form layout it will make our code-base much more optimized and easy for us to do critical form changes at a one place only.

Lets work on it, create a sample project as bellow, you can name it whatever you like.

$ laravel new crud-app

Then configure the database in .env file and run migration to create a default user table provided by laravel.

$ php artisan migrate

and create the following controller file. app/Http/Controller/UserController.php

<?php

namespace App\Http\Controllers;

use App\User;
use Illuminate\Http\Request;
use App\Http\Controllers\Controller;

class UserController extends Controller
{
    public function index()
    {
        $data['users'] = User::paginate(10);
        return view('users.index', $data);
    }

    public function create()
    {
        return view('users.add', [ 'user' => New User ]);
    }

    public function edit(User $user)
    {
        return view('users.edit', compact('user'));
    }
}

I am purposely not explaining the code for UserController.php as you can have your own code into that, we will more concentrating on views. also we are using laravel's default User Model file.

lets create a folder to the path /resources/views/users

in that folder create a three files

  • add.blade.php
  • edit.blade.php
  • form.blade.php

I am using bootstrap as a css framework, you can use whatever you want. here is the code you need put into files respectively.

add.blade.php

@extends('layout')

@section('content')
<div class="container-fluid">
    <div class="row">
        <div class="col-lg-8">
            <h1 class="page-header">Users</h1>

            <ol class="breadcrumb">
                <li><a href="{{ route('users.index') }}">Users</a></li>
                <li class="active">Add User</li>
            </ol>

            <div class="panel panel-default">
                <div class="panel-heading">Add User</div>

                <div class="panel-body">
                    <form action="{{ route('users.store') }}" method="post">
                      @include('users.form')
                    </form>
                </div>
            </div>
        </div>
    </div>
</div>
@endsection

edit.blade.php

@extends('layout')

@section('content')
<div class="container-fluid">
    <div class="row">
        <div class="col-lg-8">
            <h1 class="page-header">Users</h1>

            <ol class="breadcrumb">
                <li><a href="{{ route('users.index') }}">Users</a></li>
                <li class="active">Edit User</li>
            </ol>

            <div class="panel panel-default">
                <div class="panel-heading">Edit User</div>

                <div class="panel-body">
                    <form action="{{ route('users.update', ['user'=> $user->id]) }}" method="post">
                        {{ method_field('PUT') }}

                        @include('users.form')
                    </form>
                </div>
            </div>
        </div>
    </div>
</div>
@endsection

and finally form.blade.php

{{ csrf_field() }}

<div class="form-group{{ $errors->has('name')? ' has-error': '' }}">
    <label for="name">Name</label>
    <input type="text" name="name" class="form-control" value="{{ old('name', $user->name) }}">
    {!! $errors->has('name') ? '<span class="help-block">'.$errors->first('name').'</span>' : '' !!}
</div>

<div class="form-group{{ $errors->has('email')? ' has-error': '' }}">
    <label for="email">Email</label>
    <input type="text" name="email" class="form-control" value="{{ old('email', $user->email) }}">
    {!! $errors->has('email') ? '<span class="help-block">'.$errors->first('email').'</span>' : '' !!}
</div>

<div class="form-group{{ $errors->has('password')? ' has-error': '' }}">
    <label for="password">Password</label>
    <input type="password" name="password" class="form-control">
    {!! $errors->has('password') ? '<span class="help-block">'.$errors->first('password').'</span>' : '' !!}
</div>

<hr>

<div class="pull-right">
    <a type="button" href="{{ route('users.index') }}" class="btn btn-default">Cancel</a>&nbsp;&nbsp;
    <input type="submit" value="Submit" class="btn btn-primary">
</div>

as you can see we are not extending our layout in form.blade.php, because it is a partial view. and we are including this partial view in add.blade.php and edit.blade.php respectively, also we are binding user model object through user controller and providing a default value for old() helper function which will allow us to retrieve the old input if the validation is failed and if it does not having any value from old input then it will show values from user collection, how cool is that and the only thing you have to do is to provide empty user object with add.blade.php, just to avoid the laravel exceptions.

also one thing important I am using the method spoofing (Since HTML forms can't make PUT, PATCH, or DELETE requests, you will need to add a hidden _method field to spoof these HTTP verbs.) in edit.blade.php as {{ method_field('PUT') }}, if you are using resource routing, you can avoid this step if your are writing your own routes.

so That's all folk, we haven't created anything new but we learned to optimize our forms to make it more reusable, and you can also use this trick for large complex forms.
