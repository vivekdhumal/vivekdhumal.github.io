Make paginator component for ajax pagination using Vue js & Laravel

July 22, 2017

Okay, for this project we are using Laravel 5.4 and vue 2.4 and I assumed that you have a basic knowledge vue and laravel.

So first thing first, let's create a api controller for user feeds like so

app/Http/Controllers/Api/UserController.php

<?php

namespace App\Http\Controllers\Api;

use App\User;
use Illuminate\Http\Request;
use App\Http\Controllers\Controller;

class UserController extends Controller
{
    /**
     * Show all users.
     *
     * @return  json
     */
    public function index()
    {
        return User::paginate(5);
    }
}

Make sure to update your database connection details under the .env file.

Here we are using default user model and database migrations to create the user tables, for this will run the following command

$ php artisan migrate

Will populate the user table with some 20 fake records by using default model factory (database/factories/ModelFactory.php) provided by laravel and tinker.

$ php artisan tinker
>> factory(App\User::class, 20)->create();

and our api route file will look like

<?php

Route::group(['namespace' => 'Api'], function(){

    Route::get('/users', 'UserController@index');
});

For frontend will use the default js boilerplate code comes with laravel installation, will install the npm dependencies.

$ npm install

it will install all dependencies along with the vue and laravel-mix

let's jump to the resources director, I personaly like the ecmascript 2015 standard so first will modify the bootstrap.js like so.

import Vue from 'vue';
import Axios from 'axios';
import JQuery from 'jquery';
import Lodash from 'lodash';

window._ = Lodash;
window.Vue = Vue;

try {
    window.$ = window.jQuery = JQuery;
    require('bootstrap-sass');
} catch (e) {}

window.axios = Axios;
window.axios.defaults.headers.common['X-Requested-With'] = 'XMLHttpRequest';

let token = document.head.querySelector('meta[name="csrf-token"]');
if (token) {
    window.axios.defaults.headers.common['X-CSRF-TOKEN'] = token.content;
} else {
    console.error('CSRF token not found: https://laravel.com/docs/csrf#csrf-x-csrf-token');
}

// Creating custom event dispatcher
window.Event = new class {
    constructor() {
        // You can only listen and fire event through new vue instance
        this.vue = new Vue();
    }

    fire(event, data = null) {
        this.vue.$emit(event, data);
    }

    listen(event, callback) {
        this.vue.$on(event, callback);
    }
};

we are just modifying it as per the ECMAScript 2015 standard and we are adding a custom event dispatcher window.Event class around the vue's eventing system, to fire and listen the event easily from any vue component. you can find the reference here about the events in vue.

Then will create our ajax pagination component, Paginator.vue under the components directory like so.

<template>
    <ul class="pagination" v-if="resultSet.total && resultSet.total > 0">
        <li :class="{ 'disabled': resultSet.prev_page_url == null }">
            <a href="javascript:;" @click="goToPage(1)">&lt;&lt;</a>
        </li>

        <li :class="{ 'disabled': resultSet.prev_page_url == null }">
            <a href="javascript:;" @click="paginatePrevData()">&lt;</a>
        </li>

        <li v-for="page in resultSet.last_page" :class="{ 'active': page == resultSet.current_page }">
            <a href="javascript:;" @click="goToPage(page)">page</a>
        </li>

        <li :class="{ 'disabled': resultSet.next_page_url == null }">
            <a href="javascript:;" @click="paginateNextData()">&gt;</a>
        </li>

        <li :class="{ 'disabled': resultSet.next_page_url == null }">
            <a href="javascript:;" @click="goToPage(resultSet.last_page)">&gt;&gt;</a>
        </li>
    </ul>
</template>

<script>
    export default {
        props: [
        'resultSet'
        ],

        methods: {
            goToPage(page) {
                this.changePage(this.resultSet.path+"?page="+page);
            },

            paginateNextData() {
                this.changePage(this.resultSet.next_page_url);
            },

            paginatePrevData() {
                this.changePage(this.resultSet.prev_page_url);
            },

            changePage(url) {
                axios.get(url)
                .then(response => {
                    Event.fire('onPageChanged', response.data);
                })
                .catch(error => {
                    console.log(error.response.data);
                });
            }
        }
    }
</script>

Under the scripts tag, first will define the props for paginator component as resultSet to accept our entire json collection and access its keys to generate the html pagination component. let's create few methods under the method closure

  • changePage(url) - it will accept the pagination url and using axios to perform the ajax request to your api and under the response callback will fire the onPageChanged event to transfer the updated data to the root component where we are listening this event.
  • paginatePrevData() - it will perform the ajax request to the previous page url.
  • paginateNextData() - it will perform the ajax request to the next page url.
  • goToPage(page) - it will perform the ajax request to the specified page number.

As you can see here we are using axios library to handle ajax http requests and responses across the project, it is kind a promise based http client.

Under the template will design the bootstrap based pagination using un-ordered list and add the click events to each pagination link to make it reactive. also from api we are using some essential json objects such as next_page_url, prev_page_url, path, total, last_page, current_page to build our pagination links. also using v-for directive to generate page number, also we are using vue's class binding and attribute binding concept to highlight the link and make it disabled if we got the url as null from the api respectively.

Then will create a Users.vue component to the components directory and also add the <paginator> component to it like so.

<template>
    <div class="container">
        <div class="row">
            <div class="col-md-10 col-md-offset-1">
                <div class="panel panel-default">
                    <div class="panel-heading">Users Dashboard</div>

                    <div class="panel-body">
                        <div>
                            <a href="javascript:;" class="btn btn-sm btn-default"><i class="fa fa-plus"></i> Create New</a>
                            <br>
                            <br>
                        </div>

                        <table class="table table-striped table-borderless">
                            <thead>
                                <tr>
                                    <th>Name</th>
                                    <th>Email</th>
                                    <th></th>
                                </tr>
                            </thead>

                            <tbody>
                                <tr v-for="user in users.data">
                                    <td v-text="user.name"></td>
                                    <td v-text="user.email"></td>
                                    <td class="pull-right">
                                        <a href="javascript:;" title="edit" class="btn btn-sm btn-default"><i class="fa fa-pencil"></i></a>&nbsp;&nbsp;
                                        <a href="javascript:;" title="delete" class="btn btn-sm btn-danger"><i class="fa fa-trash"></i></a>
                                    </td>
                                </tr>
                            </tbody>
                        </table>

                        <!-- Paginator component -->
                        <paginator :result-set="users" ></paginator>
                    </div>
                </div>
            </div>
        </div>
    </div>
</template>

<script>
    export default {
        data() {
            return {
                users: [],
            }
        },

        mounted() {
            this.getAllUsers();
        },

        created() {
            /* fire ajax pagination event */
            Event.listen('onPageChanged', (data) => {
                this.users = data;
            });
        },

        methods: {
            getAllUsers() {
                axios.get("/api/users")
                .then(response => {
                    this.users = response.data;
                });
            }
        }
    }
</script>

Here we are showing the users data in table format.

Under the script tag, first of all we are defining users as empty array object under the data() method, then we define our own getAllUsers() method under the methods closure and calling it on mounted() method of our root component.

Here we will bind the users object with paginator component's result-set property to paginate our users data and we are listening the custom onPageChanged event on created method of root component to refresh the user data each time we navigate through the pagination links under the independent paginator component.

Also as you can see under the template we are iterating the user data using v-for directive and binding the column values with table cells using v-text directive.

Then we will do some refactoring in app.js, here we will register our paginator component and users component with vue, it will look like as below.

import './bootstrap';

/* Ajax-Pagination component */
Vue.component(
    'paginator',
    require('./components/Paginator.vue')
);

Vue.component(
    'users',
    require('./components/Users.vue')
);

const app = new Vue({
    el: '#app'
});

Now everything is almost done, the only thing is we need to run a laravel-mix which uses webpack to generate our final assets which will be in pure javascript form. Also me using the default webpack.mix.js from the root without changing any code that comes with the default laravel installation.

$ npm run dev

It will generate the final assets under css or js folder in public directory as you can see we are not touching the webpack.mix.js at the root and keep that file as it is, but you can modify it as per your requirement.
