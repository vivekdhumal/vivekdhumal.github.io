Laravel - How to handle exception rendering for pure json api

July 17, 2017

If you are writing a pure json api which you can consume it across the different platforms then you have to be very careful about the exception rendering.

You cannot have html based exception page with some status code (eg. 404/403), means some how you have to reproduce it in a pure json format, well with laravel it is very easy.

I ll show you how, lets consider our api endpoint is to GET the json data for task who having id 12 http://localhost:8000/api/task/12/edit and im sure most of you pretty familiar with this kind of RESTfull routes. also we are using here laravel route-model binding technique to retrieve the data.

routes/api.php

<?php

Route::get('tasks/{task}/edit', function (App\Task $task) {
    return $task;
})

If our database contains expected row of task details for id 12 then it will just simply return the json data like so.

{
    task_id: 12,
    task_name: "Bring milk and fruits",
    is_done: false,
    created_at: "2017-07-01 01:28:05",
    updated_at: "2017-07-01 01:28:05",
}

But if its not there then it will display a ModelNotFound Exception page and as we want to write a pure json api, we cannot have this exception page our response should be like as below.

{
    status: "error",
    message: "No data found",
}

also with a 404 status code at header, to handle this exception page is very easy with laravel, just go to the app/Exceptions/Handler.php and in that render method just add the following code and see the response.

    /**
     * Render an exception into an HTTP response.
     *
     * @param    \Illuminate\Http\Request  $request
     * @param    \Exception  $exception
     * @return  \Illuminate\Http\Response
     */
    public function render($request, Exception $exception)
    {
        if($exception instanceOf \Illuminate\Database\Eloquent\ModelNotFoundException) {
            if ($request->expectsJson()) {
                return response()->json(['status' => 'error', 'message' => 'No data found.'], 404);
            }
        }
        return parent::render($request, $exception);
    }

Here we are checking first is Exception object is instance of ModelNotFoundException and if its true then we are checking if our request is expecting json or not and if it does then we will respond with json data, thats it. I hope this will help you.
